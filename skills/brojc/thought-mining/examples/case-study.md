# Thought Mining 案例：从20个碎片到一篇深度文章

## 案例背景

用户想要写一篇关于 "Claude Skills" 的文章，但脑子里只有一堆零散想法：
- "Skills 是 Prompt 升级"
- "不是万能的"
- "和 GPTs 不一样"
- "让 Agent 更全能"
- "需要配置"
- "是工作流"
- ...

共20多个碎片，不知道怎么串起来。

---

## 阶段1：思维挖掘

**AI**：关于 Claude Skills这个话题，你脑子里最先蹦出来的是什么？随便说。

**用户**：我觉得 Skills 是 Prompt 的升级版，但又不太一样。Prompt 是一次性的，Skills 更像是...一个完整的工作流？能记住上下文，还有自己的知识库。

**AI**：为什么这么说？能举个例子吗？

**用户**：比如我写了个"写周报"的 Skill，它知道我的工作内容，帮我整理每周做了什么，还知道用什么格式。这比单纯给一个 prompt "帮我写周报" 强多了。

**AI**：还有呢？

**用户**：Skills 不是万能的。很多人以为有了 Skills 就不用学编程了，不是的。你还是得知道怎么调教它。而且 Skills 和 GPTs 不一样，GPTs 是封闭的，Skills 是开放的...

（经过多轮追问，收集到20+洞察点）

---

## 阶段2：选题确定

**AI**：好的，你有很多想法了。如果只能选一个角度写，你会选哪个？

**用户**：嗯...Skills 是工作流+知识库？让 Agent 更全能？

**AI**：这几个角度好像都有人写了。有没有更独特的视角？

**用户**：那... Skills 说明书？

**AI**：展开说说？

**用户**：你想啊，没有 Skills 的 Agent，就像一个聪明但容易忘事的实习生。你得天天叮嘱它。但 Skills 就像——是那个实习生的说明书。一次性写好，下次直接用。

**AI**（标注金句）：这句话太棒了！没有 Skills 的 Agent，是个聪明但容易忘事的实习生——Skills 是前者的说明书。

如果只能告诉别人一句话，你会说什么？

**用户**：就是这句：**没有 Skills 的 Agent，是个聪明但容易忘事的实习生。**

---

## 阶段3：观点验证

**AI**：我去查了一下 Claude Skills 的官方定义...和你说的基本一致。它确实是"可复制的 Agent 能力封装"，支持自定义指令和知识库。

关于"和 GPTs 不一样"这个点，GPTs 是 OpenAI 的封闭生态，Skills 是开放的，这个对比也站得住。

整体验证通过，信心指数：**高**

---

## 阶段4：写作辅助

**用户**（开始写）：Claude Skills 是 Prompt 的升级...

**AI**（阶段4反馈）：这段写得不错，"升级"这个词很形象。但要注意，Skills 不只是"更长的 Prompt"，它有工作流和知识库两个核心能力，是不是可以区分一下？

**用户**：对，我重写一下：Skills = 工作流 + 知识库...

**AI**：这个概括清晰。最后那段总结可以更有力，现在有点平。

---

## 阶段5：最终审核

**审核结果**：

| 检查项 | 状态 |
|--------|------|
| 主题清晰 | ✅ 一句话就能说清楚 |
| 逻辑通顺 | ✅ 从定义到差异到价值，链条完整 |
| 事实正确 | ✅ 验证通过 |
| 表达流畅 | ✅ "实习生vs说明书"是亮点 |
| 标题吸引人 | ✅ 可优化，建议："没有 Skills 的 Agent，就像..." |

**最终发布建议**：可以发！

---

## 关键总结

1. **阶段1不评判**：用户说"Skills是Prompt升级"时，AI没有纠正，而是继续深挖
2. **阶段2关键转折**：通过"一句话"测试找到了文章题眼
3. **阶段3必要且高效**：几分钟验证，节省了可能出错的风险
4. **阶段4持续跟进**：边写边反馈，比写完再改效率高
