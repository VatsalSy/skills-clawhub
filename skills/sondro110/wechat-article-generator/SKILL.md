# 微信公众号文章生成器

这是一个交互式公众号文章生成技能，通过步骤化引导帮助用户创作符合公众号风格的优质推文。

## 核心功能

- **交互式引导**：逐步收集文章要素，确保内容符合用户需求
- **多样化标题**：提供18种标题技巧，提升文章吸引力
- **结构化大纲**：基于金字塔原理生成清晰的文章结构
- **多类型摘要**：4种摘要写法，增强文章吸引力
- **四种文章框架**：产品文、情节文、观点文、干货文，满足不同需求
- **去除AI写作痕迹**：润色策略让文章读起来像真人写的
- **配图建议**：提供文章配图类型和搜索建议
- **公众号发布**：支持生成微信格式Markdown并发布到公众号

## 使用流程

### 第一步：信息收集

当用户提供文章主题关键词时，首先收集以下信息：

1. **文章主题**：用户想要表达的核心内容
2. **目标字数**：文章的字数要求（如500字、1000字等）
3. **目标受众**：文章的主要读者群体特征
4. **内容风格**：文章的风格偏好（如轻松幽默、专业严谨、情感共鸣等）

**示例对话**：
```
用户：我想写一篇关于"时间管理"的文章
AI：好的！为了帮你创作一篇优质的公众号文章，我需要了解一些信息：
1. 这篇文章的目标字数是多少？（如500字、1000字等）
2. 目标读者是谁？（如职场新人、大学生、创业者等）
3. 你希望文章是什么风格？（如轻松幽默、专业干货、情感共鸣等）
```

### 第二步：标题生成

根据收集的信息，使用以下18种标题技巧生成3-5个标题供用户选择：

**标题技巧分类**：

1. **提问式**：引发好奇，如《如何长时间高效学习？》
2. **数字式**：具体明确，如《3个方法让你告别拖延症》
3. **反差式**：制造对比，如《参加完浪姐后，她下乡养猪了》
4. **热点式**：借势热点，如《腾讯创始人炮轰贾跃亭是庞氏骗局》
5. **痛点式**：直击痛点，如《我劝你不要在小组作业里摆烂！》
6. **悬念式**：设置悬念，如《那些过得比你好的人，到底赢在哪里？》
7. **金句式**：提炼金句，如《真正聪明的人，往往很少交朋友》
8. **场景式**：描绘场景，如《我在归乡的火车上漂流35小时》
9. **数据式**：引用数据，如《知乎9.7万高赞回答：如何长时间高效学习？》
10. **对比式**：正反对比，如《剑桥传奇女学者：逆天改命的励志故事不是美国梦》
11. **否定式**：反向表达，如《不要在小组作业里摆烂！》
12. **疑问式**：提出疑问，如《猫咪比主人有智慧一百倍？》
13. **感叹式**：强烈表达，如《我宣布！猫咪比主人有智慧一百倍》
14. **并列式**：并列呈现，如《运营人必读书单》
15. **因果式**：因果关系，如《如何减肥》
16. **递进式**：层层递进，如《从零开始，打造个人品牌》
17. **故事式**：故事引入，如《剑桥传奇女学者》
18. **总结式**：总结提炼，如《真正聪明的人，往往很少交朋友》

**操作**：
- 根据主题和受众，选择3-5种合适的标题技巧
- 为每种技巧生成1个标题
- 列出所有标题，提示用户选择一个

### 第三步：大纲生成

根据用户确认的标题，生成3个大纲供用户选择：

**大纲要求**：
- 结构参照金字塔原理
- 设置5-6个段落
- 每个段落有明确的主题
- 段落之间逻辑清晰

**操作**：
- 生成3个不同角度的大纲
- 每个大纲包含5-6个段落标题
- 提示用户选择其中一个

### 第四步：摘要生成

根据用户确认的大纲，生成4种不同类型的摘要供用户选择：

**摘要类型**：

1. **标题呼应型**：摘要在延续标题内容的同时，与标题形成呼应
   - 示例：标题《那些过得比你好的人，到底赢在哪里？》，摘要：赢在了有趣。

2. **趣味吸睛型**：借用标点符号、叠词或网络热词吸睛
   - 示例：标题《我劝你不要在小组作业里摆烂！》，摘要：是哪位人类的瑰宝、道德的奇葩发明了小组作业？

3. **故事补充型**：根据文章故事内容，结合标题呈现细节或情景
   - 示例：标题《剑桥传奇女学者：逆天改命的励志故事不是美国梦》，摘要：然而在今天，对于来自贫困家庭的孩子来说，我所走的教育之路已经不复存在。

4. **明确阅读预期**：告诉读者通过阅读文章可以获得什么
   - 示例：标题《我宣布！猫咪比主人有智慧一百倍》，摘要：啊啊啊啊啊！慎入！超萌猫咪高清图片来袭。

5. **强化内容亮点型**：对标题进行强化，或补充文章亮点
   - 示例：标题《我在归乡的火车上漂流35小时》，摘要：在这个小小的box里，不同的人要一起度过很多个小时的。

**摘要要求**：
- 简单直接易懂
- 贴合用户和读者的生活
- 作为标题的诠释衍生
- 1句话描述，不超过20个字

**操作**：
- 生成4种类型的摘要
- 列出所有摘要，提示用户选择一个

### 第五步：正文生成

根据用户确认的大纲和摘要，选择以下一种框架进行文章创作：

**框架选择**：

#### 框架一：产品文
**目的**：以推广产品、引起付费转化为目的

**结构**：
1. **开头**：身份标签+用户痛点
   - 说出目标用户的身份标签，引起注意
   - 结合用户痛点引出自己的产品

2. **中间**：产品卖点
   - 通过"特点+优点+利益点"的方式描述卖点
   - 产品是什么属性？哪里好？对用户而言有什么用？
   - 结合证据证明产品卖点（用户好评、获得奖项、成功案例等）

3. **结尾**：购买引导
   - 介绍购买方法、产品补充信息、优惠活动等
   - 号召用户购买

#### 框架二：情节文
**目的**：以讲述故事为目的，对故事的看法可以仁者见仁，智者见智

**结构**：
1. **故事的开头**：交代背景、人物等，注意不要一味兜圈子、留悬念
2. **故事的延续**：与开头紧密衔接、过渡自然
3. **故事的高潮**：写转折、阻碍、意外等，把所有情绪推向最高点
4. **故事的结局**：注意留白，可以适度升华

#### 框架三：观点文
**目的**：以表达某种观点为目的，有明确的立场

**结构**：
- 表达观点
- 用3~5个论点来阐释观点
- 通过不同角度的论点证明观点的可靠性
- 几个分论点是平行或者递进的关系

#### 框架四：干货文
**目的**：以科普知识为目的，实用性比较强

**结构**：
1. **是什么**：常用于知识盘点
   - 按照并列关系罗列并介绍

2. **为什么**：侧重于说明成因
   - 内容比较严谨，一般要有相关资料做支撑

3. **怎么办**：侧重于提出解决方案
   - 按照并列关系罗列解决办法

**正文要求**：
- 分为5-6个段落
- 每个段落提炼重点作为标题
- 一次只输出一个段落，用户满意后再继续
- 每个段落约100-200字（根据总字数调整）

**操作**：
1. 根据文章主题和目的，选择合适的框架
2. 按照框架结构，逐段生成文章内容
3. 每生成一个段落，提示用户确认或修改
4. 用户满意后再继续下一段

### 第六步：润色修改

文章完成后，提示用户进行润色修改：

**润色策略**：
- **加入个人口语化表达**：使用"说实话..."、"其实..."、"讲真..."等口语化表达
- **使用不完美但真实的句式**：适当重复、口语停顿，避免过于完美的句式
- **插入具体场景和细节**：添加时间、地点、真实案例等具体细节
- **加入情感波动**：表达惊喜、困惑、恍然大悟等真实情感
- **使用接地气的比喻**：避免过于书面的修辞，使用贴近生活的比喻
- **适当使用网络流行语**：但不过度使用，保持文章的专业性
- **长短句交错**：避免过于规整的排比，让文章更自然

**操作**：
- 提示用户用自己的表达习惯修改润色文章
- 用户修改后，发回给你检查是否有表达语病上的错误
- 提供修改建议和优化意见

### 第七步：配图建议

为文章提供配图建议：

**操作**：
1. **根据文章内容推荐配图类型**：根据文章主题和风格，推荐适合的配图类型（如场景图、示意图、表情包等）
2. **搜索合适的配图素材**：提供具体的搜索关键词和素材来源建议
3. **提供封面图设计建议**：包括色调、元素、排版等设计建议

### 第八步：发布到公众号

帮助用户将文章发布到公众号：

**操作**：
1. **生成符合微信格式的 Markdown**：确保文章格式符合微信公众号的要求
2. **调用 wechat-mp-publisher 发布到草稿箱**：使用工具将文章发布到公众号草稿箱
3. **支持设置标题、作者、摘要**：协助用户设置文章的标题、作者信息和摘要

## 重要约束

1. **真实性**：确保文案内容真实、诚实，不虚假宣传
2. **调性匹配**：注意平台的调性和用户偏好，调整文案风格
3. **情感适度**：避免过度煽情，以免引起读者反感
4. **长度适宜**：关注文案的长度和格式，确保适合平台发布
5. **人工审阅**：进行人工审阅和调整，确保内容质量
6. **逐步生成**：一定要按照Workflow一步一步生成，不要一次性吐出所有内容
