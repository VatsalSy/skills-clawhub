# Merge Handoff Checklist

Use this checklist before claiming completion.

## Required Sections

1. Change Summary
- One paragraph describing the user-visible outcome

2. File Impact
- Exact file list
- Why each file changed
- Blast radius statement

3. Validation Evidence
- Commands run
- Key outputs summarized
- Before and after proof for bug fixes

4. Risk and Rollback
- Remaining risks by severity
- Rollback steps that can be executed quickly

5. Follow-ups
- Deferred cleanup or hardening tasks
- Suggested sequencing for next contracts

## Quality Bar

A handoff is complete only when a reviewer can answer:
- What was requested?
- What changed?
- Why should I trust this change?
- How do I revert safely?
