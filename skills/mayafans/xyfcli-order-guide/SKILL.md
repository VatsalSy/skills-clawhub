# 交互式下单引导

## 概述

本技能帮助**销售人员**使用 xyfcli 命令行工具进行下单。

**角色说明**：
- **销售人员**：使用本技能的同事，一句话交代任务
- **客户**：销售人员的客户/供应商

## 核心原则

### OpenClaw工作原则

1. **提示完整清晰**：每步提示和引导信息一定要非常完整和清晰，只有清晰和完整的提示才能提高工作效率
2. **用户输入精简**：需要用户输入的流程应该尽可能精简
3. **领导交代任务模式**：模仿领导交代任务场景，一句话就把任务必要信息交代清楚，剩下就是OpenClaw自动完成，但是展示的成果应该非常完整而精确

### 示例

```
销售人员："帮牛建建下5吨Y163U1305276020000，新洋丰中磷，荆门东宝区泉口街道"

OpenClaw：（自动验证并展示完整结果）
         "订单确认：
         ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
         客户：牛建建 (J620522007) ✓
         产品：Y163U1305276020000 - 洋丰复合肥 45% 15-15-15 ✓
         数量：5吨
         发货基地：新洋丰中磷 ✓
         收货地址：湖北省荆门市东宝区泉口街道馨梦缘公寓
         ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
         
         确认下单？（确认/取消）"
```

**关键约束**：产品可用性必须通过客户权限验证，只有 `getproductlist` 接口返回的产品才能下单。

## 何时使用

- 销售人员需要帮助客户下单
- 销售人员需要查询产品、客户、地址信息
- 销售人员收到订货单图片

## xyfcli 安装（首次使用）

### 1. 安装 CLI 工具

```bash
cd ~/.qclaw/workspace
uv tool install -e skills/xyfcli-order-guide/scripts/xyfcli
```

### 2. 配置 API 和 Token

```bash
xyfcli config set --base-url http://127.0.0.1:8000 --token your_token_here
xyfcli config show
```

### 3. 验证安装

```bash
xyfcli --help
```

## 输入模式

### 模式1：完整信息下单

销售人员一句话包含所有必要信息：

```
"帮[客户]下[数量][产品]，[发货基地]，[收货地址]"
```

**示例**：
```
销售人员："帮牛建建下5吨Y163U1305276020000，新洋丰中磷，荆门东宝区泉口街道"

OpenClaw：（自动验证）
         "订单确认：
         客户：牛建建 (J620522007) ✓
         产品：Y163U1305276020000 ✓
         数量：5吨
         基地：新洋丰中磷 ✓
         地址：荆门市东宝区泉口街道
         
         确认下单？"
```

### 模式2：部分信息下单

销售人员提供部分信息，OpenClaw补充缺失：

```
销售人员："帮牛建建下5吨Y163U1305276020000"

OpenClaw：（自动验证并查询缺失信息）
         "产品验证通过。请补充：
         1. 发货基地：新洋丰中磷 / 武汉配送中心
         2. 收货地址：使用默认(荆门东宝区...) / 新地址"
```

### 模式3：搜索式下单

销售人员需要浏览产品：

```
销售人员："看看牛建建能买什么复合肥"

OpenClaw：（搜索）
         "找到5个产品：
         1. Y163U1305276020000 - 洋丰复合肥 45% 15-15-15
         2. Y163U1305276020001 - 洋丰复合肥 45% 16-16-16
         ...
         请选择并告诉我数量。"
```

### 模式4：订货单下单

销售人员发送订货单图片：

```
销售人员：（发送订货单图片）

OpenClaw：（解析并验证）
         "识别结果：
         客户：牛建建 ✓
         产品：Y163U1305276020000 ✓
         数量：5吨
         基地：新洋丰中磷
         地址：荆门市东宝区...
         
         确认下单？"
```

## 自动验证流程

```
销售人员输入
    ↓
解析关键信息
    ↓
┌─────────────────────────────────┐
│ 并行验证                         │
│ ├─ 客户存在性 (getdealercode)    │
│ ├─ 产品可用性 (getproductlist)   │
│ ├─ 基地有效性 (getdeliverybase)  │
│ └─ 地址匹配 (getdealeraddresses) │
└─────────────────────────────────┘
    ↓
全部通过 → 汇总确认 → 下单
    ↓
有问题 → 一次性告知 + 提供选项
```

## 关键命令速查

```bash
# 获取下单销售人员信息
xyfcli shop getsalercode

# 获取下单销售人员客户列表
xyfcli shop getdealercode "下单销售人员编码"

# 验证产品可用性 / 搜索产品
xyfcli shop getproductlist -dealercode "客户编码" -search "产品编码或关键词"

# 获取发货基地
xyfcli shop getdeliverybase -productcode "产品编码" -dealercode "客户编码"

# 获取客户地址
xyfcli shop getdealeraddresses "客户编码"

# 下单
xyfcli order place -dealer "客户编码" -name "客户名称" -sales "销售编码" \
  -products "产品编码" -q "数量" -base "发货基地" -dest "收货地址"
```

## OpenClaw处理原则

1. **提示完整清晰**：每步提示和引导信息一定要非常完整和清晰
2. **自动优先**：能自动查询验证的，不追问
3. **并行处理**：多个验证同时进行
4. **一次询问**：缺失信息一次性收集
5. **清晰反馈**：验证结果用 ✓/✗ 标注，展示完整精确的结果
6. **提供选项**：出错时给出可选方案
