# 知识图谱更新规则

## 掌握程度标记

| 等级 | 含义 | 标准 |
|------|------|------|
| 🔴 入门 | 听说过，知道是什么 | 能说出一句话定义 |
| 🟡 熟悉 | 读过源码或论文，能解释原理 | 能向别人讲清楚核心机制 |
| 🟢 精通 | 有实战经验，能独立设计和排错 | 有代码贡献或项目实战，踩过坑并解决 |

## 升级判断

升级前问自己：

### 🔴 → 🟡
- **我能不看资料向别人解释这个概念吗？**
- **我知道它的核心实现原理吗？**（不是"大概知道"，是能说出关键数据结构和算法）
- **我读过相关源码或论文的核心部分吗？**

### 🟡 → 🟢
- **我有真实的代码贡献或项目实战吗？**（PR、项目、实验）
- **我踩过坑并自己解决了吗？**
- **如果别人遇到这个领域的问题来问我，我能给出具体建议吗？**

## 降级情况

- 超过 3 个月没接触且无法回忆核心细节 → 考虑降级
- 技术发生重大变化，之前的理解已过时 → 重新评估

## 更新格式

在 `ai-knowledge-map.md` 的升级日志中记录：

```
| 日期 | 主题 | 变更 | 原因 |
| 2026-02-16 | MCP 协议 | 🔴→🟡 | 读完 spec + 提了 3 个相关 PR |
```
