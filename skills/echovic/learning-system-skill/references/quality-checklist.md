# 交付质量检查清单

深度笔记或复盘完成后，逐项检查：

## 内容质量
- [ ] 核心概念用自己的话解释了，不是复制粘贴文档
- [ ] 有具体的源码分析或技术细节，不是泛泛而谈
- [ ] 包含"为什么"而不只是"是什么"（设计决策、trade-off）

## 关联完整性
- [ ] 至少建立了 1 个关联（`→ 关联: [主题](路径)`）
- [ ] 关联到了知识图谱中的具体领域
- [ ] 如果有对比关联（A vs B），说清了关键差异

## 实战价值
- [ ] 包含实战关联（与自己的项目/PR 的关系）
- [ ] 有"开放问题"或"深入方向"，指向下一步
- [ ] 如果是复盘，记录了踩坑和解决过程

## 知识图谱同步
- [ ] 相关主题的掌握程度已更新（如有变化）
- [ ] 升级日志已记录变更原因
