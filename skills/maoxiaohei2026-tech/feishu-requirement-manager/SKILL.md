# 需求管理 Skill (v1.3.0)

## ⚡ 核心流程

```
用户: 创建需求 "XXX"
    ↓
创建需求记录
    ↓
分配给「任务拆分Agent」分析需求
    ↓
「任务拆分Agent」返回任务列表
    ↓
创建任务记录
    ↓
分配给合适的执行Agent（搜索/文档/分析Agent）
    ↓
异步执行任务 → 完成 → 通知用户
```

---

## 📋 Agent 职责分工

| Agent | 职责 |
|-------|------|
| 主Agent | 协调者：接收指令、调用工具、更新状态、通知用户 |
| 任务拆分Agent | 分析需求、智能拆分为可执行任务、分配执行人 |
| 搜索Agent | 执行搜索信息类任务 |
| 文档Agent | 执行文档创建/编辑类任务 |
| 分析Agent | 执行数据分析类任务 |

---

## 🎯 完整功能流程

### 1. 创建需求
**触发**: 用户说"创建需求：XXX"

**执行**:
1. 解析需求信息
2. 创建需求记录，状态="待处理"
3. 触发任务拆分

---

### 2. 任务拆分（由拆分Agent执行）
**触发**: 创建需求后自动触发，或用户说"拆分需求"

**执行流程**:

#### Step 1: 调用任务拆分Agent
```json
sessions_spawn({
  "task": "分析以下需求，拆分为具体可执行的任务：\n\n需求内容：{需求标题}\n{需求描述}\n\n请返回任务列表，格式：\n1. 任务标题\n2. 任务描述\n3. 推荐执行人（搜索Agent/文档Agent/分析Agent/用户）\n4. 依赖关系（如有）",
  "label": "task-split-{需求ID}",
  "runtime": "subagent",
  "mode": "run"
})
```

#### Step 2: 接收拆分结果
拆分Agent返回任务列表，包含：
- 任务标题
- 任务描述
- 推荐执行人
- 依赖关系

#### Step 3: 创建任务记录
```
feishu_bitable_app_table_record.batch_create({
  records: [
    {"fields": {"任务标题": "xxx", "执行人": "搜索Agent", "任务状态": "待处理"}},
    ...
  ]
})
```

#### Step 4: 更新需求状态
```
需求状态 → "进行中"
```

---

### 3. 任务执行（异步）
**触发**: 任务创建后自动触发

**执行流程**:

#### Step 1: 查询待执行任务
```
feishu_bitable_app_table_record.list({
  filter: {
    "conditions": [
      {"field_name": "任务状态", "operator": "is", "value": ["待处理"]}
    ]
  }
})
```

#### Step 2: 按依赖顺序分配给对应Agent
根据"执行人"字段分配：

| 执行人 | 调用的Agent |
|--------|------------|
| 搜索Agent | sessions_spawn(search task) |
| 文档Agent | sessions_spawn(document task) |
| 分析Agent | sessions_spawn(analysis task) |
| 用户 | 通知用户处理 |

#### Step 3: 执行并更新状态
```
任务状态: "进行中" → "已完成"
需求进度: 已完成任务/总任务数 × 100%
```

#### Step 4: 全部完成
```
需求状态 → "已完成"
通知用户
```

## 📖 完整示例

### 示例: 用户说"创建需求：分析AI发展趋势报告"

```
Step 1: 创建需求
  - 需求标题：分析AI发展趋势报告
  - 状态：待处理
  - 优先级：P1

Step 2: 调用任务拆分Agent
  sessions_spawn({
    task: "分析需求'分析AI发展趋势报告'，拆分为任务",
    runtime: "subagent"
  })

Step 3: 拆分Agent返回
  - 任务1：搜索AI最新动态（搜索Agent）
  - 任务2：整理行业报告（分析Agent）
  - 任务3：撰写报告（文档Agent）

Step 4: 创建任务记录
  - 任务1：搜索AI动态，执行人=搜索Agent，状态=待处理
  - 任务2：整理行业报告，执行人=分析Agent，状态=待处理
  - 任务3：撰写报告，执行人=文档Agent，状态=待处理

Step 5: 异步执行任务
  - 搜索Agent执行任务1 → 完成
  - 分析Agent执行任务2 → 完成
  - 文档Agent执行任务3 → 完成

Step 6: 全部完成
  - 需求状态=已完成
  - 通知用户
```

---

## ⚠️ 注意事项

1. **拆分Agent** - 专门负责分析需求和拆分任务
2. **执行Agent** - 根据任务类型分配给合适的Agent
3. **异步执行** - 使用sessions_spawn确保不阻塞
4. **状态更新** - 每个任务完成后立即更新状态和进度
