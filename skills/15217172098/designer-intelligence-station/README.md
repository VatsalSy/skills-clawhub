## 📖 技能概述

**设计师情报站**是一个专为设计师打造的情报收集与筛选**分析框架**，覆盖 AI、智能硬件、手机、设计四大领域。

### 核心定位

本技能是一个**分析框架 + 执行指南**，而非独立运行的自动化工具。它提供：

1. **情报筛选方法论** - 5 维设计师视角筛选标准
2. **输出模板** - 日报/周报结构化格式
3. **执行流程** - 情报收集、筛选、分析的完整步骤
4. **配置指南** - 监测源列表、cron 任务配置示例

### 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    用户/Agent 执行环境                        │
├─────────────────────────────────────────────────────────────┤
│  1. 监测源配置 (用户本地配置)                                  │
│     └─→ memory/intelligence-monitor-config.md               │
│                                                              │
│  2. 情报抓取 (Agent 调用 web_fetch/browser 工具)              │
│     └─→ 访问配置的监测源 URL                                  │
│                                                              │
│  3. 情报筛选 (本技能提供的 5 维标准)                           │
│     └─→ 按筛选条件过滤，标注条件编号                           │
│                                                              │
│  4. 报告生成 (使用本技能提供的模板)                           │
│     └─→ temp/intelligence-daily-template.md                 │
│     └─→ temp/intelligence-weekly-template.md                │
│                                                              │
│  5. 定时推送 (用户配置 cron 任务)                             │
│     └─→ OpenClaw cron 调度器                                  │
└─────────────────────────────────────────────────────────────┘
```

### 为什么这样设计？

**透明性原则**: 本技能不封装黑盒逻辑，所有配置、模板、筛选标准都公开可见，用户可完全掌控：

- ✅ 监测源可自定义（增删改查）
- ✅ 筛选标准可调整（5 维标准可修改）
- ✅ 推送时间可配置（cron 表达式）
- ✅ 输出格式可定制（模板文件可编辑）

## 📋 核心功能

### 1. 5 维筛选标准（核心方法论）

每条情报必须满足以下**至少一个**条件，否则不予收录：

| 维度 | 权重 | 说明 | 示例 |
|------|------|------|------|
| **[1] 知名大厂动态** | ⭐⭐⭐ | 头部公司战略级产品/技术发布 | 苹果发布会、谷歌 I/O、华为主题日 |
| **[2] 趋势洞察** | ⭐⭐⭐ | 反映新技术方向、交互创新 | AI 设计工具、新交互范式、多模态体验 |
| **[3] 设计哲学启发** | ⭐⭐ | 帮助理解品牌理念、设计方法论 | 苹果设计访谈、IDEO 方法论、品牌重塑案例 |
| **[4] 竞品策略分析** | ⭐⭐ | 揭示竞品设计策略、体验差异 | 同类产品设计对比、体验拆解、设计系统更新 |
| **[5] 设计判断力提升** | ⭐⭐⭐ | 深度分析文章、趋势报告 | 设计趋势报告、用户行为研究、前瞻性分析 |

**排除内容**：
- ❌ 纯市场宣传（新闻稿、广告）
- ❌ 常规版本更新（小版本号迭代）
- ❌ 融资八卦（未经证实的传闻）
- ❌ 参数对比（跑分、规格表）
- ❌ 营销话术（"颠覆"、"革命性"等空洞表述）

### 2. 情报分级机制

| 等级 | 判定标准 | 处理方式 |
|------|----------|----------|
| **S 级** | 满足 2 个以上条件 + 大厂发布 | 头条位置 + 深度分析 |
| **A 级** | 满足 2 个条件 | 重点报道 |
| **B 级** | 满足 1 个条件 | 简讯收录 |
| **C 级** | 不满足任何条件 | 排除 |

### 方式三：配置自动化推送（高级用户）

#### 步骤 1：创建监测源配置文件

在 `memory/intelligence-monitor-config.md` 配置监测源列表（参考上方"监测源配置"部分）。

#### 步骤 2：配置 cron 定时任务

**日报推送**（每日 19:00）:
```json
{
  "name": "设计师情报日报",
  "schedule": {
    "kind": "cron",
    "expr": "0 19 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "请执行每日情报汇总任务：\n\n【筛选标准】每条情报必须满足以下至少一个条件：\n1. 知名大厂动态\n2. 趋势洞察\n3. 设计哲学启发\n4. 竞品策略分析\n5. 设计判断力提升\n\n【执行步骤】\n1. 读取 memory/intelligence-monitor-config.md\n2. 抓取监测源最新资讯\n3. 按筛选标准过滤\n4. 按照 temp/intelligence-daily-template.md 格式生成日报"
  },
  "delivery": {
    "mode": "announce",
    "to": "你的用户 ID",
    "channel": "你的渠道"
  }
}
```

**周报推送**（每周一 9:00）:
```json
{
  "name": "设计师情报周报",
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * 1",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "请执行每周情报汇总任务：\n\n【筛选标准】（同上）\n\n【执行步骤】\n1. 汇总上周（周一至周日）资讯\n2. 按筛选标准过滤\n3. 按照 temp/intelligence-weekly-template.md 格式生成周报"
  },
  "delivery": {
    "mode": "announce",
    "to": "你的用户 ID",
    "channel": "你的渠道"
  }
}
```

#### 步骤 3：测试推送

```bash
# 手动触发测试
cron run <任务 ID>
```

## ❓ FAQ

**Q: 这个技能是自动运行的吗？**

A: 不是。本技能是一个分析框架和执行指南，需要用户（或 Agent）手动触发或配置 cron 定时任务来执行。

**Q: 我需要配置 API 密钥吗？**

A: 不需要。本技能使用公开信息源，无需任何 API 密钥。如需访问付费内容，需用户自行配置。

**Q: 监测源可以自定义吗？**

A: 可以。在 `memory/intelligence-monitor-config.md` 中自由增删改监测源。

**Q: 推送时间可以修改吗？**

A: 可以。修改 cron 任务的 `expr` 字段即可调整推送时间。

**Q: 输出格式可以定制吗？**

A: 可以。编辑 `temp/intelligence-daily-template.md` 和 `temp/intelligence-weekly-template.md` 自定义输出格式。

**Q: 为什么有些情报没有被收录？**

A: 本技能采用严格的 5 维筛选标准，排除纯市场宣传、常规版本更新、融资八卦、参数对比、营销话术等低质量内容。筛选通过率通常在 10-20%。

**Q: 支持其他语言吗？**

A: 当前主要支持中文。英文内容会自动翻译摘要。

## 🎯 最佳实践

### 1. 监测源配置

- **少而精**: 优先选择高质量信息源，避免信息过载
- **定期清理**: 每季度检查监测源有效性，移除失效链接
- **领域平衡**: 确保 AI、手机、设计、硬件四大领域覆盖均衡

### 2. 筛选标准执行

- **严格执行**: 不满足任何条件的情报坚决排除
- **条件标注**: 每条收录情报标注满足的条件编号
- **S 级判定**: 仅当满足 2+ 条件且为大厂发布时定为 S 级

### 3. 输出质量控制

- **来源标注**: 每条情报标注来源，便于追溯
- **设计师视角**: 趋势洞察必须包含设计师视角的分析和建议
- ** actionable**: 建议必须具体可执行，避免空洞表述

### 4. 自动化配置

- **测试先行**: 正式配置 cron 前先手动测试
- **超时设置**: 日报 timeout 设为 300 秒，周报设为 600 秒
- **错误处理**: 配置失败时保留错误日志
